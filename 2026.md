# üåü Happy New Year 2026: Habit Tracker Experience

A minimalist, high-end web experience designed to celebrate the New Year while promoting healthy daily consistency. This project combines smooth CSS animations, modern glassmorphism UI, and dynamic JavaScript-generated visual effects.

---

## üìñ What it Does
This project is a visual storytelling piece. Instead of a static greeting, it features a **sequenced reveal** of text that guides the user through essential daily habits:
* **Running & Physical Activity**: Encouraging 10k steps.
* **Health & Nutrition**: Reminders for hydration and clean eating.
* **Consistency**: The final message‚Äî*"Small habits. Big years."*‚Äîemphasizes the power of daily discipline.

The page uses a dynamic "Glitter Engine" to create an immersive, twinkling background that feels like a deep-space environment.

---

## üõ†Ô∏è What it Uses (Tech Stack)



* **HTML5**: Semantic structure for the landing page.
* **CSS3 (Advanced)**: 
    * **Glassmorphism**: Uses `backdrop-filter: blur(12px)` for a frosted-glass effect.
    * **Custom Keyframes**: Specialized `@keyframes` for the `twinkle` and `reveal` animations.
    * **Flexbox**: Ensuring the UI is perfectly centered and responsive across all screens.
* **JavaScript (ES6)**: 
    * **Dynamic Injection**: A loop that creates 50+ sparkle elements on the fly.
    * **Math Randomization**: Every sparkle has a randomized position, size, and animation delay to ensure no two visits look the same.
* **Typography**: Integrated with Google Fonts (**Inter** font family).

Since this project is built with **Vanilla Web Technologies**, there are no dependencies to install and no build steps required.

1.  **Clone the Repository**:
    ```bash
    git clone [https://github.com/sarthaksm29-blip/HTML-CSS-assignments.git](https://github.com/sarthaksm29-blip/HTML-CSS-assignments.git)
    ```
2.  **Navigate to the folder**:
    ```bash
    cd HTML-CSS-assignments
    ```
3.  **Launch the Page**:
    * **Option A**: Simply double-click the `index.html` file to open it in your browser.
    * **Option B (Recommended)**: If using VS Code, right-click `index.html` and select **"Open with Live Server"** for the best experience.
